# IDS 705 Final Project: World Cup Analytics

This repository contains Team 9's IDS 705 final project on forecasting men's FIFA World Cup match outcomes. The main modeling task predicts match results from the perspective of the higher-Elo team:

- `fav_wins`: the higher-Elo team wins
- `draw`: the match ends in a draw
- `fav_loses`: the higher-Elo team loses

The final report uses the multiclass modeling pipeline as the main evidence base, with additional binary-classification and regression experiments included as supplementary work.

## Data

All data needed to reproduce the submitted results is included in this repository.

The data is organized into four layers.

### 1. Raw source layer

These files preserve the original match, team, ranking, roster, and player-performance inputs.

- `data/raw/worldcup/data-sqlite/worldcup.db`
  - Core World Cup match database.
  - Used for matches, teams, tournaments, squads, managers, and historical match outcomes.
- `data/raw/elo_ratings.csv`
  - National team Elo ratings.
  - Used to identify the pre-match favorite and build team-strength features.
- `data/raw/fifa_ranking-2024-06-20.csv`
  - FIFA ranking snapshots.
  - Used as a second national-team strength signal.
- `data/raw/player_rosters.csv`
  - World Cup squad lists.
  - Used in player and squad feature engineering.
- `data/raw/player_stats_prior_season.csv`
  - Prior-season club performance for World Cup players.
  - Used to build player-informed features.
- `data/raw/club_elo_ratings.csv`
  - Club Elo ratings.
  - Used to attach club-strength context to player records.
- `data/raw/teams.csv`
  - Team metadata and confederation labels.
  - Used for joins and regional reliability analysis.

### 2. Team-level feature layer

These files add match history, tournament context, Elo, FIFA ranking, manager, rest-day, and squad-demographic features.

- `data/processed/class_a_match_level.csv`
  - Base match-level feature table.
- `data/processed/class_a_with_rankings.csv`
  - Match-level table with Elo and FIFA ranking features.
- `data/final/class_a_with_rankings_2006_2022.csv`
  - Final 2006 to 2022 team-plus-ranking table.

### 3. Player and squad feature layer

These files summarize player club performance, club strength, squad composition, and squad-level aggregates.

- `data/processed/player_stats_with_club_elo.csv`
  - Player-level table with prior-season club statistics and club Elo context.
- `data/processed/squad_level_features.csv`
  - Squad-level aggregates by tournament and team.
- `data/processed/class_a_with_squad_features.csv`
  - Match-level table after adding squad features.

### 4. Final modeling table layer

This is the main merged table for final model development and comparison.

- `data/final/class_a_squad_2006_2022.csv`
  - Final 2006 to 2022 match-level table with team, ranking, and squad features.
  - This is the clearest starting point for downstream modeling checks.

The final consolidated notebook `notebooks/model/worldcup_multiclass_model.ipynb` rebuilds the multiclass modeling table from the raw World Cup database, ranking files, and `data/processed/player_stats_with_club_elo.csv`. The final CSV files in `data/final/` provide the cleaned modeling tables used by the broader project and supplementary notebooks.

## Setup

Use Python 3.11 or newer.

Create and activate a clean environment:

```bash
python -m venv .venv
source .venv/bin/activate
```

Install dependencies:

```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Register the environment as a Jupyter kernel:

```bash
python -m ipykernel install --user --name ids705-worldcup --display-name "ids705-worldcup"
```

Start Jupyter:

```bash
jupyter notebook
```

When opening notebooks, choose the `ids705-worldcup` kernel.

## Final Modeling Summary

The final report compares three multiclass systems:

| Model | Training window | Feature scope | Final model family |
|---|---:|---|---|
| Baseline | 2006 to 2014 | Team-level features | Random forest |
| Team-history | 1998 to 2014 | Team-level features | Random forest |
| Player-informed | 2006 to 2014 | Team plus player/squad features | XGBoost |

The main conclusion is cautious. The team-history model is the safest general forecasting anchor. The player-informed model has the strongest macro-F1 point estimate and looks most useful as a second view for knockout and upset-risk analysis, but its advantage is uncertain and sensitive to algorithm choice.
